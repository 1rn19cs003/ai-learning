LLMs from the Ground Up → Advanced RAG

An exhaustive, self-paced curriculum with a flagship portfolio project
Tailored for a JavaScript/TypeScript engineer (2+ yrs) moving into AI engineering

How to use this document. Work through milestones strictly in order — each checkpoint gates the next milestone. Estimated total: 14–18 weeks at 8–10 hrs/week. Code in TypeScript wherever the ecosystem allows (noted per milestone); Python appears only where it is unavoidable (training internals, some eval tooling). The portfolio project at the end is built incrementally: each milestone ends with a project task, so by the final milestone you have shipped the whole system.

Part I — Foundations: how LLMs actually work ✎ notes

MILESTONE 0 · ~1 week

Math prerequisites (only what you actually need)

What you will learn

  • Linear algebra: vectors, matrices, matrix multiplication as transformation, dot products as similarity — the single most important intuition for everything that follows.
  • Calculus: derivatives, the chain rule, gradients — enough to understand backpropagation, no more.
  • Probability: distributions, conditional probability, softmax as a distribution over tokens, cross-entropy loss.

Resources

  • 3Blue1Brown — Essence of Linear Algebra (video series; watch all of it, it's the best visual intuition available)
  • 3Blue1Brown — Essence of Calculus (chapters 1–4 suffice)
  • Mathematics for Machine Learning (free book — use as reference, not cover-to-cover)
Implement, in plain TypeScript with no libraries: dot product, matrix multiply, softmax, and cosine similarity. Then explain in writing (5 sentences max each) why softmax turns logits into probabilities and why cosine similarity measures semantic closeness.
Expected: ~60 lines of TS, and you can answer "why divide by the norm?" without looking it up.
MILESTONE 1 · ~1.5 weeks

Neural networks & backpropagation from scratch

What you will learn

  • Neurons, weights, biases, activation functions; why depth and nonlinearity matter.
  • Backpropagation as recursive chain rule; gradient descent, learning rates, loss landscapes.
  • Autograd — how PyTorch/TensorFlow compute gradients automatically.

Resources

  • Karpathy — "The spelled-out intro to neural networks and backpropagation: building micrograd" (the canonical from-scratch lecture)
  • karpathy/micrograd (repo — ~100 lines of autograd)
  • 3Blue1Brown — Neural Networks series (visual companion)
Port micrograd to TypeScript. Train a tiny MLP on XOR and on a 2D spiral dataset; plot the decision boundary (Canvas or a simple HTML page — you're a JS dev, use it).
Expected: your TS autograd trains XOR to ~0 loss, and you can explain what a single .backward() call does step by step.
MILESTONE 2 · ~1 week

Tokenization & embeddings — how text becomes numbers

What you will learn

  • Byte-Pair Encoding (BPE), WordPiece, SentencePiece; vocabulary size trade-offs; why "strawberry has two r's" failures happen; why tokenization drives API cost.
  • Static embeddings (word2vec: skip-gram, CBOW) → contextual embeddings; the king − man + woman = queen geometry.
  • Embedding matrices as the first layer of every LLM.

Resources

  • Karpathy — "Let's build the GPT Tokenizer" (build BPE from scratch)
  • word2vec paper — "Efficient Estimation of Word Representations" (Mikolov et al., 2013)
  • Jay Alammar — The Illustrated Word2vec
  • openai/tiktoken and the tokenizer playground (paste text, watch it split)
  • JS: dqbd/tiktoken (WASM bindings for Node/browser)
Implement BPE training in TypeScript on a small corpus (e.g., a book from Project Gutenberg). Show the learned merges. Then build a small web page where typing text shows live token counts using tiktoken-WASM.
Expected: your BPE produces sensible subwords, and you can explain why "internationalization" is ~4 tokens but "the" is 1.
MILESTONE 3 · ~2 weeks (the core)

The Transformer & attention, in full depth

What you will learn

  • Scaled dot-product attention: Q, K, V matrices — what each projection means; why divide by √dk; causal masking.
  • Multi-head attention: why multiple heads, what different heads learn.
  • The full decoder block: attention → residual → LayerNorm → MLP → residual; why residual streams matter.
  • Positional information: sinusoidal encodings vs. learned vs. RoPE (rotary) — what modern models actually use.
  • The KV cache — why generation is fast after the first token, and why long contexts eat memory.

Resources

  • "Attention Is All You Need" (Vaswani et al., 2017 — read after the visual resources, not before)
  • Jay Alammar — The Illustrated Transformer (start here)
  • Karpathy — "Let's build GPT: from scratch, in code, spelled out" (the single best resource in this curriculum)
  • karpathy/nanoGPT (repo to read line by line)
  • RoFormer / RoPE paper (Su et al., 2021)
  • bbycroft.net/llm — interactive 3D visualization of a running transformer (built in JS, incidentally)
Follow Karpathy's video and build nanoGPT in Python (this is the one milestone where Python is genuinely the right tool — PyTorch has no TS equivalent). Train a character-level model on Shakespeare. Then, separately, implement single-head scaled dot-product attention in pure TypeScript for a fixed toy input and verify the output against a hand computation.
Expected: generated Shakespeare-ish text, and you can whiteboard the attention formula and say what every symbol does.
MILESTONE 4 · ~1 week

How models are trained: pretraining → instruction tuning → alignment

What you will learn

  • Pretraining: next-token prediction at scale; data pipelines; scaling laws (Kaplan, Chinchilla) — why model and data size trade off.
  • Supervised fine-tuning (SFT) and instruction tuning: how a text-completer becomes an assistant.
  • RLHF and DPO: reward models, preference optimization; what "alignment" concretely means.
  • Parameter-efficient fine-tuning: LoRA/QLoRA — the low-rank trick, when fine-tuning beats RAG (rarely) and when it doesn't.

Resources

  • Scaling Laws for Neural Language Models (Kaplan et al., 2020)
  • Chinchilla — "Training Compute-Optimal LLMs" (Hoffmann et al., 2022)
  • InstructGPT paper (Ouyang et al., 2022 — the RLHF blueprint)
  • DPO — "Direct Preference Optimization" (Rafailov et al., 2023)
  • LoRA paper (Hu et al., 2021)
  • Karpathy — "Intro to Large Language Models" (1-hour big-picture talk)
  • Hugging Face NLP Course (chapters on fine-tuning)
Write a one-page explainer (as if to another JS dev) of the full journey: raw internet text → base model → SFT → RLHF/DPO → the chat model you call via API. Include where LoRA fits and one concrete case where you would fine-tune instead of using RAG.
Expected: a peer could read it and correctly answer "why does the base model complete text instead of answering questions?"

Part II — Using LLMs & the geometry of meaning ✎ notes

MILESTONE 5 · ~1 week

Working with LLMs as an engineer TS-first

What you will learn

  • Sampling: temperature, top-p, top-k — what each knob actually changes in the softmax distribution.
  • Prompt engineering as engineering: system prompts, few-shot, chain-of-thought, XML/structured delimiting, JSON mode / structured outputs.
  • Function/tool calling: schemas, parallel calls, the request→tool_use→tool_result loop.
  • Streaming (SSE), retries, rate limits, token budgeting and cost math.

Resources

  • Anthropic — Prompt engineering docs
  • Anthropic — Tool use docs
  • Vercel AI SDK docs (TypeScript; streaming, tool calling, structured output)
  • anthropic-sdk-typescript · openai-node
  • Ollama (run local models; has a JS client) — compare a local 8B model vs. a frontier API model on the same prompts
Build a TS CLI/mini-web app: streams responses, enforces a JSON schema output with retry-on-invalid, exposes temperature/top-p sliders, and logs tokens + cost per request. Demonstrate one tool call (e.g., a calculator the model invokes).
Expected: you can predict, before running, how changing temperature from 0 to 1.2 alters output for a factual vs. creative prompt.
MILESTONE 6 · ~1 week

Embeddings at scale & approximate nearest-neighbor search TS-first

What you will learn

  • Modern embedding models (OpenAI text-embedding-3, Cohere embed, open-source bge/e5); Matryoshka embeddings and dimension trade-offs.
  • Similarity metrics: cosine vs. dot vs. Euclidean — when normalization makes them equivalent.
  • Why exact search dies at scale; ANN algorithms: HNSW (graph-based) and IVF (clustering) — how they work, recall/latency trade-offs.
  • Vector databases: Qdrant, pgvector, Pinecone, Chroma — index types, metadata filtering, when Postgres+pgvector is all you need.

Resources

  • HNSW paper (Malkov & Yashunin, 2016)
  • Pinecone Learning Center (excellent free series on vector search & ANN)
  • Sentence-BERT paper (Reimers & Gurevych, 2019)
  • Qdrant docs (JS client) · pgvector
  • MTEB leaderboard (how embedding models are benchmarked)
In TypeScript: embed ~5k paragraphs, implement brute-force cosine search yourself, then stand up Qdrant (Docker) and compare — recall@10 of ANN vs. brute force, and latency of both at 5k and 50k vectors.
Expected: a small table of numbers you measured, and you can explain what HNSW's "layers" buy you.

Part III — RAG, from naive to state of the art ✎ notes

MILESTONE 7 · ~1 week

RAG fundamentals — and why naive RAG fails TS-first

What you will learn

  • The core RAG loop: retrieve top-k → stuff into context → generate; why RAG beats fine-tuning for knowledge injection (freshness, provenance, cost).
  • Chunking: fixed-size vs. recursive vs. semantic vs. structure-aware (headings, code blocks); overlap; chunk-size trade-offs.
  • Failure modes to induce deliberately: lost-in-the-middle, chunk-boundary splits, vocabulary mismatch, multi-hop questions naive RAG cannot answer.
  • Citations and grounding: making the model quote its sources.

Resources

  • The original RAG paper (Lewis et al., 2020)
  • "Lost in the Middle" (Liu et al., 2023)
  • LangChain RAG tutorial · LangChain.js docs · LlamaIndex.TS
  • Anthropic — Contextual Retrieval (prepending chunk context before embedding; large accuracy gains, simple idea)
Build a naive RAG pipeline in TS over ~50 markdown/PDF docs. Then write 10 questions where it fails, categorize each failure (retrieval miss? chunking? generation?), and keep this list — it becomes your M9 eval set.
Expected: a working pipeline AND a documented failure taxonomy. The failures are the real deliverable.
MILESTONE 8 · ~2 weeks

Advanced RAG: hybrid, transformation, re-ranking, Graph RAG TS-first

What you will learn

  • Hybrid search: BM25 (the math: TF-IDF → BM25 saturation) + dense vectors; Reciprocal Rank Fusion (RRF).
  • Query transformation: rewriting, decomposition, multi-query expansion, HyDE (embed a hypothetical answer), step-back prompting.
  • Re-ranking: bi-encoder vs. cross-encoder — why cross-encoders are more accurate and too slow for first-stage; two-stage retrieval.
  • Contextual retrieval: LLM-generated chunk context at index time.
  • Graph RAG: entity/relation extraction → knowledge graph → community summaries; when graphs beat vectors (multi-hop, "how does X relate to Y" questions).
  • Self-correcting retrieval: CRAG (grade retrievals, fall back to web search) and Self-RAG (model critiques its own groundedness).

Resources

  • HyDE paper (Gao et al., 2022)
  • Cohere Rerank docs · bge-reranker (open source)
  • GraphRAG paper (Edge et al., Microsoft 2024) · microsoft/graphrag (repo)
  • CRAG — Corrective RAG (Yan et al., 2024)
  • Self-RAG paper (Asai et al., 2023)
  • Weaviate — Hybrid Search Explained
  • NirDiamant/RAG_Techniques (repo — dozens of RAG patterns with runnable notebooks; the best single RAG repo to study)
Upgrade your M7 pipeline: add BM25 + RRF fusion, multi-query expansion, and a cross-encoder reranker. Re-run your 10 failure questions — document which failures each technique fixed. Then build a minimal Graph RAG index over the same corpus and find one question only the graph can answer.
Expected: a before/after table per technique. If you can't name which technique fixed which failure, you added them blindly — go back.

Part IV — Evaluation, agents, production ✎ notes

MILESTONE 9 · ~1 week

Evaluation frameworks — Ragas, TruLens, promptfoo

What you will learn

  • The RAG triad (TruLens): context relevance, groundedness, answer relevance.
  • Ragas metrics: faithfulness, answer relevancy, context precision, context recall — how each is computed with LLM-as-judge, and the pitfalls of LLM judges.
  • Building golden datasets; synthetic test-set generation; regression testing prompts like code.
  • Tracing and observability: spans for every retrieve/rerank/generate step.

Resources

  • Ragas docs (Python — worth the small detour; concepts pages explain each metric's math)
  • TruLens docs (the RAG triad)
  • promptfoo docs (TS-native eval harness — your CI tool)
  • Langfuse docs (tracing; first-class JS SDK)
Turn your M7 failure list plus 20 new Q&A pairs into a versioned eval set. Wire promptfoo to score faithfulness and relevance on every run, and Langfuse to trace every pipeline stage. Produce one chart: naive RAG vs. advanced RAG scores on the same set.
Expected: a reproducible npm run eval that fails if faithfulness drops below a threshold — evaluation as CI, not as a one-off notebook.
MILESTONE 10 · ~1.5 weeks

Agents, routing & multi-agent orchestration TS-first

What you will learn

  • The ReAct loop (reason → act → observe) implemented by hand — no framework black boxes.
  • Routing: classify the query, send it to the right retriever/model/pipeline; semantic routers.
  • Multi-agent patterns: orchestrator–worker, critic/reviewer loops, when multiple agents genuinely beat one (and when they're hype).
  • State machines for agents (LangGraph.js); handling loops, budgets, and failure recovery.

Resources

  • ReAct paper (Yao et al., 2022)
  • Anthropic — Building Effective Agents (the essential essay on when NOT to use agents)
  • LangGraph.js docs
  • Vercel AI SDK — agents
Implement a ReAct agent in raw TypeScript (no framework) with three tools: your RAG pipeline, a calculator, and web search. Add a critic step that grades answers for groundedness and triggers one retry. Then rebuild it in LangGraph.js and write 5 sentences on what the framework bought you.
Expected: visible reasoning traces, a bounded loop (max N steps), and an informed opinion on frameworks vs. hand-rolled.
MILESTONE 11 · ~1 week

Production engineering TS-first

What you will learn

  • Streaming architectures end to end (server → SSE → React UI); optimistic UI for agent steps.
  • Prompt caching (90% cost cuts on repeated context); semantic response caching; model routing (cheap model first, escalate on low confidence).
  • Incremental re-indexing; document-change detection; index versioning.
  • Guardrails: input/output filtering, prompt-injection defenses for RAG (retrieved content is untrusted input!).
  • Deployment: Docker, serverless trade-offs for long-running agent loops, monitoring with your M9 eval suite as a canary.

Resources

  • Anthropic — Prompt caching docs
  • Simon Willison — prompt injection series (required reading)
  • Chip Huyen's blog (production ML/LLM systems essays)
  • FastAPI if you ever need Python services; otherwise Hono / Next.js API routes
Containerize the full system. Add prompt caching and measure the cost delta on 50 queries. Write one prompt-injection attack that hijacks your own RAG pipeline via a poisoned document, then patch it.
Expected: a docker-compose that boots everything, a real cost number, and a working exploit + fix — that last one is a fantastic interview story.

Flagship Portfolio Project — RepoSage

A self-correcting, multi-agent codebase intelligence engine

The pitch: point RepoSage at any Git repository and it becomes the senior engineer who has been on the team for years. It answers not just "where is X?" but "why is X built this way?" — by retrieving over three synchronized indexes: the code itself (AST-aware), the git history (commits, blame, PR descriptions), and the documentation (including architecture diagrams, parsed multimodally). A router agent picks retrieval strategies, a critic agent grades every answer for groundedness before it ships, and failed retrievals trigger self-correcting query rewrites (CRAG-style). This is not chat-with-your-PDF — it is multi-index hybrid retrieval + Graph RAG over code relationships + multi-agent orchestration + multimodal ingestion + a full eval harness, wrapped in a polished streaming React UI (your JS advantage, weaponized).

Why this project stands out

  • Nobody's demo does "why". Answering "why was this pattern chosen?" requires joining code + git archaeology + docs — three retrieval modalities fused. That's a systems story, not a tutorial clone.
  • Self-correction is visible. The UI shows the critic rejecting an answer and the router retrying — reviewers see the advanced architecture working.
  • It's demoable on any public repo — an interviewer can name a repo and watch it work live.
  • Every curriculum topic is load-bearing: tokenization (cost budgeting), attention (context ordering), embeddings/ANN, hybrid + Graph RAG, agents, evals, production hardening.
